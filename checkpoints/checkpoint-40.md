# Checkpoint 40 — reference-link wiring + dhatu stubs (Slice C of v14)

**Date:** 2026-05-10

## Trigger

User comparison: `तस्य` (a pronoun) showed *"see in सर्वनाम — तद्-template (master सर्वनाम)"* as a clickable reference at the bottom of its popover. Most other words showed nothing equivalent.

> *"Some words like तस्य have reference attached to it (here `see in सर्वनाम — तद्-template (master सर्वनाम)`). Can we have the same for missing ones like हर्षम्, विनद्य etc. (from 1.12)? Ideally I would want it for all the items, all the words in पदच्छेद."*

The mechanism existed (`getDeclensionForParsing` for nouns, `getPronounAnchor` for pronouns), but only fired for those two categories — verbs, krdantas, particles, and suffix-inferred entries dumped into the same EmptyPopover.

Then a second category emerged. User clicked "open √ध्मा in Verbs page" from the प्रदध्मतुः popover → silently landed on √भू (the first dhātu) because √ध्मा wasn't in DHATUS_EXTENDED. The fallback was masking real failures.

> *"But when I click to go on the verbs page, I don't see any ध्मा dhatu... I would like to go to the actual verb root to figure out what is what. This way I am dumped to the primer which isn't much useful."*

## What changed

### `src/data/referenceLinks.js` — generic reference-link helper (`f08186a`)

`getReferenceLink(parsing)` returns `{ url, label }` for every category:

| Category | Link target |
|---|---|
| noun / adjective | paradigm-class if classifiable (देव/फल/गुरु/आत्मन्/कर्मन्/मनस्/सीता/मति); else `/atlas/declensions` |
| pronoun | tad-template anchor if classifiable; else `/atlas/pronouns` |
| krdanta absolutive | `/primer#krdanta` — *"see in Primer — कृदन्त (absolutive form)"* |
| krdanta infinitive / gerundive | `/primer#krdanta` — kind-labelled |
| krdanta past-passive | paradigm if has gender; else Primer कृदन्त |
| krdanta present-active (शतृ) | `/primer#krdanta` |
| verb | `/verbs/<root>` if dhatu in DHATU_REGISTRY; else `/primer#lakara` with explicit "(not in top-192 list yet)" notice |
| particle | `/atlas/avyaya` |
| numeral | `/primer#vibhakti` |

`WordPopover` renders the generic link as a fallback when the existing paradigm and pronoun-anchor branches don't already produce one. Every popover now has at least one outbound pedagogical link.

For the user's 1.12 example:
- `हर्षम्` (with explicit gender=m added) → "follows देव-class — see all 24 forms"
- `विनद्य` → "see in Primer — कृदन्त (absolutive form)"
- `दध्मौ` → "open √ध्मा in Verbs page"
- `उच्चैः` → "see in Atlas — अव्यय (indeclinables)"

### `Verbs.jsx` getDhatuById matches by id OR devanagari (`aee7cb4`)

Was: `DHATUS.find((d) => d.id === id) ?? null` — pure id match.
Is: `id` first, then `devanagari` (with or without trailing virama). So `/verbs/ध्मा` resolves correctly when the dhatu IS in the list.

### `_dhatus_extra.js` — 49 stub entries (`aa27ec0`)

Auto-generated by walking SHARED_VOCAB + CORE_VOCAB + VOCAB_EXTENDED for verb roots that aren't in the curated top-192 list. Each stub has: id (devanagari, no virama) · devanagari · meanings (up to 3 from gloss text) · gana · pada · thematic flag · note ("Stub from vocab evidence: <sample-forms>").

Notable additions: **√ध्मा (blow), √जन् (be born), √ब्रू (speak), √चर् (move), √बन्ध् (bind), √भाष् (speak), √रभ् (begin), √हा (give up), √कथ् (tell), √हृष् (rejoice), √ईक्ष् (see / regard), √व्यथ् (be agitated), √जागृ (be awake), √काङ्क्ष् (desire), √भास् (illumine), √आस् (worship / sit), √आप् (obtain), √आह् (defective "say"), √सू (bring forth), √विज् (be agitated), √व्रज् (walk)** + 28 others. Composed into DHATUS_EXTENDED via the existing mergeDhatus() pipeline.

Total dhatu count: **141 → 190**.

### `referenceLinks` consults DHATU_REGISTRY (`aee7cb4`, plus update in `aa27ec0`)

Built once at module load (Set of all 190 ids + devanagari forms). The `verb` case now checks:
1. If bare-root from `√X` extraction exists in DHATU_REGISTRY → link to `/verbs/X`.
2. If not but lakara is set → `/primer#lakara` with explicit "(√X not in the top-192 list yet)" notice.
3. If neither → `/verbs` index.

So `प्रदध्मतुः` now reads "open √ध्मा in Verbs page" → `/verbs/ध्मा` → lands on the actual root with gloss + gana + pada displayed, instead of the silent /verbs/भू fallback.

### Heuristic samas type tagging (`319195a`)

(Cross-listed in [Slice A](checkpoint-38.md) — happened in the same time window.) Two heuristics on the `samasNotes` derivation:
1. **3+ noun parts → `इतरेतर द्वंद्व (inferred)`**: coordinated lists. पणव-आनक-गोमुखाः ("drum-and-drum-and-horn"), जन्म-कर्म-फल-प्रदाम्.
2. **2 noun parts → `तत्पुरुष? (inferred — best guess)`**: most common 2-stem compound.

### लङ् patterns (`997f8a8`)

User flagged `अभ्यहन्यन्त` (1.13 — passive imperfect 3pl, "they-were-struck / sounded forth") showing "no grammar data". Form starts with अ- augment + ends in -न्त (no virama after त) — ātmanepada or passive imperfect 3pl. Three patterns added to the suffix-fallback:
- `/^अ.+न्त$/` → लङ् 3pl ātmanepada/passive
- `/^अ.+त्$/` → लङ् 3sg
- `/^अ.+न्$/` → लङ् 3pl परस्मैपद

Plus explicit entries for अभ्यहन्यन्त and अहन्यन्त. Both now resolve with proper category=verb, lakara=lan, and the "open √हन् in Verbs page" reference link.
