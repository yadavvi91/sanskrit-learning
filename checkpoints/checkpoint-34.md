## How

Six lanes, each split into 3 chapter-range part files (ch 1–6 / 7–12 / 13–18) generated by parallel agents, composed by a tiny wrapper module:

```
src/data/
  translations-besant.js    — 676 entries
  translations-arnold.js    — wraps _arnold_part{1,2,3}.js
  translations-hindi.js     — wraps _hindi_part{1,2,3}.js
  interpretive.js           — wraps _interp_part{1,2,3}.js
  hydrate.js                — runs at module init, fills auto-stub
                              verses from these maps
```

Hydrator is gated behind `if (v.tier !== 'auto-stub') continue` and per-field `!v.<field>` checks, so the 25 hand-decoded verses (1.1, 2.3, 2.4, 2.5, plus the 21 v10 backfill verses) are never overwritten.

The user explicitly chose **"Everything, paraphrase-flagged"** via clarifying question, accepting that this content needs auditing. The Decode Helper UI is the path to promote a verse from auto-stub → browse → full once the user has reviewed it.
