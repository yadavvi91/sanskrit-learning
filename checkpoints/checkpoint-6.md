# Checkpoint v6 — Slice 3: Stack Builder formula + reverse-decoder rework

**Date:** 2026-05-08
**Branch:** `implement-all-plans`
**Slice:** v6 refinement, slice 3

## What changed

### Forward formula

User feedback: "the bhava + t = bhavat plus should not be inside a border."

The "+" and "=" operators are no longer wrapped in `formula-piece` chips. They render as bare `formula-op` text between the bordered chips. The formula reads cleanly:

```
[ अ ] + [ भव ] + [ त् ] = [ अभवत् ]
```

instead of the prior "[ + त् ]" mistake. Added a small color-key legend underneath the formula labelling each chip's role (augment / stem / ending) using colored dots that match the chip backgrounds.

### Reverse decoder

User feedback: "I don't know how that works. I paste some word, and what do you give? You give me direct answers for that?"

Reverse mode now mirrors Forward visually — same chip-and-operator layout — instead of dumping a flat 6-line attribute list. For प्रतियोत्स्यामि:

```
[ प्रति ] + [ योत्स्य ] + [ ामि ] = [ प्रतियोत्स्यामि ]
                  लृट् (Future) · परस्मैपद · उत्तम एकवचन · गण 4
```

The उपसर्ग chip (gold border) precedes the stem chip; lakāra/pada/puruṣa/vacana/gana shrink into a single annotation row beneath. Multiple matches still surface as separate cards, but each card now reads as an *assembly*, not an attribute dump.

Default placeholder changed from अकुर्वत to प्रतियोत्स्यामि — showcases the उपसर्ग parser.

## Why

Same formula visual grammar across Forward and Reverse means once the user gets either, they get both. The free operators ("+", "=") read as math operators, the chips read as pieces; mixing the two was the original mistake.
